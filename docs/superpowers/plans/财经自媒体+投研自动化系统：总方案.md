## 财经自媒体+投研自动化系统：总方案

### 一、核心定位

> **人机协同的工程化内容与投研系统**  
> 人定方向、给观点、做终审；AI做拆解、扩写、加速；系统做沉淀、调度、优化。

---


本系统服务两个核心需求：

1. **财经自媒体自动化/半自动化**
   - 自动追踪热点并生成选题建议
   - 抓取爆款图文/视频并做转写、拆解
   - 基于选题自动生成高质量文案
   - 可选导出到公众号、小红书等平台

2. **金融投研自动化/半自动化**
   - 自动采集公开财经网站数据：行情、财务指标、公告、新闻
   - 根据用户自定义规则筛选股票/标的
   - 利用 AI 做基本面分析、情绪解读和风险提示
   - 定时执行并推送日报/盘后报告

**总原则：**
- 确定性任务优先使用代码与规则引擎
- AI 主要承担摘要、归纳、解释、生成与多文档综合推理
- 默认采取**半自动化**，尤其是外发内容与投资建议输出，保留人工审校阀门
- 优先做最小可用闭环，再做降本与自动化增强
- 所有自动化都必须纳入预算、可观测性、熔断与人工接管框架
### 二、设计哲学（你的原则，直接沿用）

| 原则 | 含义 |
|------|------|
| **工程化思维** | 内容生产模板化、参数化；投研规则化、可回测。高频确定性流程固化为代码。 |
| **精准主义** | 数据清洗+来源校验+人工抽查；AI输出必须带依据、风险提示。 |
| **Harness Engineering** | 所有模型调用经过Harness层（校验、路由、日志、降级），禁止裸调。 |
| **PDCA闭环** | 计划→执行→检查→改进，每周迭代。 |

### 三、总体架构（三层）

```
┌─────────────────────────────────────────┐
│ 调度层（crontab → Python入口）           │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│ Harness层（模型封装、日志、降级）         │

└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
```

**运行环境**：台式机（RTX 3060 + Ollama本地模型）+ 腾讯云（仅用于cron触发，可选）。

### 四、功能模块与流程

#### 1. 热点追踪（手动+自动）
- **自动**：OpenClaw 抓取财经热榜，每日推送到本地文件。
- **手动**：你筛选3-5个值得写的方向。

#### 2. 爆文拆解（每周2-3次）
- **AI**（云端模型）：拆解结构（hook类型、逻辑链、情绪强度），输出JSON。
- **人**：判断“为什么火”，标记可借鉴点，存入 `structures/` 文件夹。

#### 3. 选题生成（每日）
- **AI**（云端模型）：基于热点+历史爆文结构，生成5个选题（标题+角度+说明）。
- **人**：删除3个平庸选题，保留1-2个“你有话说的”，手工插入 `topics` 表。

#### 4. 初稿生成（每日自动）
- **Harness层**：从 `topics` 表读取未使用选题，调用本地模型（Qwen2.5-7B）生成2000字初稿。
- **输出**：Markdown文件到 `drafts/`，同时记录 trace_id 到日志。

#### 5. 人工审阅与发布（每日）
- 打开 `drafts/` 中的初稿，修改数据错误、添加个人观点、去AI味。
- 复制到公众号/知乎发布，手工记录阅读量、点赞到 `posts` 表（初期用Excel，后期可API）。

#### 6. 投研日报（每日自动）
- Python脚本调用 akshare 获取低PE+高增长股票，生成Markdown报告到 `research/daily/`。
- 可选：将报告中的股票作为选题来源，注入 `topics` 表。

#### 7. 复盘与优化（每周）
- **AI**：分析本周发布数据，找出表现最好的文章，总结可能原因。
- **人**：确认原因，调整选题方向、prompt模板或Harness规则。

### 五、降维落地版（Phase 2，立即执行）

> **目标：2周内跑通“自动选题生成+初稿”闭环，手工发布，不上复杂调度。**

| 组件 | 选型 | 理由 |
|------|------|------|
| 调度 | crontab（`0 9 * * * python run.py`） | 比n8n简单100倍，无额外故障点 |
| Harness层 | 单文件 `harness.py`，封装本地模型调用+日志 | 保留工程灵魂，但去掉路由/熔断 |
| 数据库 | SQLite，仅 `topics` 和 `posts` 两表 | 轻量，可导出Markdown查看 |
| 本地模型 | Ollama + qwen2.5:7b | 6G显存够用，免费，数据不出本地 |
| 云端模型 | GPT-4o-mini 或 DeepSeek-V2（仅拆解+选题） | 成本低，质量高 |
| 通知 | 企业微信机器人（发送“今日初稿已生成”） | 简单，免费 |
| 发布 | 手工复制到公众号 | 初期不折腾API |

**目录结构**（总代码<300行）：
```
content-system/
├── run.py              # 主入口
├── harness.py          # AI调用封装
├── local_model.py      # Ollama调用
├── db.py               # SQLite操作
├── drafts/             # 每日初稿
├── topics.db
└── logs/
```

**cron示例**：
```bash
0 9 * * * cd /home/user/content-system && python run.py >> logs/cron.log 2>&1
```

### 六、演进路线图（PDCA驱动）

| 阶段 | 时间 | 核心任务 | 自动化程度 |
|------|------|----------|------------|
| **Phase 1**（已完成） | 第1周 | 手工抓热点、拆爆文、选题，验证内容生产流程 | 仅AI辅助写作 |
| **Phase 2**（2-4周） | 第2-4周 | 实现上述降维落地版，每日自动生成初稿，手工发布 | 选题自动生成，初稿自动 |
| **Phase 3**（1-2个月） | 第5-8周 | 增加反馈回流（API获取阅读量），选题评分模型（简单线性） | 选题自动排序，人工选Top3 |
| **Phase 4**（3个月后） | 第9周+ | 引入n8n编排多任务，增加熔断、降级、多模型fallback | 全自动（高风险内容人工审） |

**关键决策点**：
- 进入Phase 3前，必须积累至少30篇发布数据，否则评分模型无效。
- 进入Phase 4前，必须确认每日任务链稳定运行2周无人工干预。

### 七、成功标准（可量化）

| 指标 | 目标 | 验收方式 |
|------|------|----------|
| 日更频率 | 每周≥3篇 | `posts` 表记录 |
| 单篇初稿生成耗时 | ≤3分钟 | 日志埋点 |
| AI初稿可用率 | ≥80%（人修改<20%） | 主观评估 |
| 投研日报准确率 | 数据无误，筛选逻辑正确 | 每周抽查2次 |
| 系统稳定性 | 连续7天无崩溃 | cron日志 |

---

### 八、一句话总结

> **用工程化思维设计，用降维方式落地。**  
> 先跑通 `cron + Harness + SQLite + 本地模型` 的最小闭环，每周基于真实数据做一次PDCA，3个月后自然长成你想要的完全自动化系统。
