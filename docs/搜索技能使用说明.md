# 搜索技能使用说明

> 目标：把 `exa-search` 与 `grok-search` 接入当前 YMOS / OpenClaw 工作区时，明确它们各自的边界，避免把“找证据”和“看动态”混成一锅。

---

## 1. 当前已安装的搜索技能

已安装到：

- `skills/exa-search`
- `skills/grok-search`

这两个 skill 是互补关系，不是替代关系：

- `exa-search`：**源头优先 / 官方文档 / 低噪声 / 正文提取**
- `grok-search`：**时效优先 / 实时动态 / 社区讨论 / 多源综述**

---

## 2. 什么时候用 `exa-search`

优先用于：

- OpenClaw / PinchTab / Skill 官方文档
- API 参数、Reference 页面
- 官网定价页、产品说明页
- 公司官网 / 项目主页 / 白皮书
- 需要从网页抽正文而不只是拿链接
- 需要尽量避开 SEO 垃圾结果时

一句话：

> **要找原文、找证据、找官方来源，用 `exa-search`。**

### 典型问题

- `OpenClaw Telegram streaming 官方文档怎么写？`
- `某工具的 pricing 页面在哪里？`
- `帮我找这个 API 参数的官方 reference`
- `只要官方来源，不要二手总结`

---

## 3. 什么时候用 `grok-search`

优先用于：

- 实时变化的话题
- 最近几天的市场讨论
- 社区舆情 / X(Twitter) 讨论 / 动态面
- “外面现在都在怎么说？”
- 快速理解一个正在变化中的事件
- 对比“官方怎么说”和“社区怎么解读”

一句话：

> **要看现在发生了什么、外面怎么讨论，用 `grok-search`。**

### 典型问题

- `最近外面对某项目怎么评价？`
- `X 上关于某币最近在讨论什么？`
- `帮我快速总结这个主题最近几天的动态`
- `官方说法和社区解读有什么差异？`

---

## 4. 在投研场景里的推荐路由

### 场景 A：验证事实

先问：**我要的是事实，还是讨论？**

如果要的是：
- 公告
- 官网
- 文档
- 参数
- 定价
- 白皮书

直接走：
- `exa-search`

### 场景 B：感知市场动态

如果要的是：
- 最近热度
- 舆论分歧
- 社区观点
- 讨论轮廓

先走：
- `grok-search`

再视情况补：
- `exa-search` 做官方核验

### 场景 C：投研标准动作（推荐）

最推荐的组合流程：

1. 用 `grok-search` 看最近发生了什么
2. 用 `exa-search` 回到官方或源头页面核实
3. 输出时区分：
   - **已核实事实**
   - **社区讨论/市场解读**

---

## 5. 哪些情况不能只信 `grok-search`

以下情形，不能只靠 `grok-search` 下结论：

- 交易决策
- 工具能力判断
- API / 参数配置
- 合约地址 / 产品定价 / 规则条款
- 任何需要“源头可追溯”的结论

原因：

- `grok-search` 更适合发现线索
- 但“线索”不等于“证据”

所以对重要结论：

> **`grok-search` 负责发现方向，`exa-search` 负责找证据。**

---

## 6. 哪些情况不能默认只信 `exa-search`

以下情形，如果只用 `exa-search`，容易漏掉实时变化：

- 热点事件
- 舆情反转
- 社区争议
- 市场情绪变化
- 刚发布的新产品/新版本的用户反馈

原因：

- `exa-search` 偏“源头与低噪声”
- 但未必最擅长“现在外面正在怎么说”

---

## 7. 当前落地建议

### 推荐默认规则

- **文档 / 参数 / 官方信息 / 正文提取** → `exa-search`
- **实时动态 / 讨论热度 / 舆情扫描** → `grok-search`
- **重要结论** → 至少做一次 `exa-search` 回源验证

### 推荐输出结构

在投研或研究回答中，尽量区分：

- **结论**
- **官方/源头证据**
- **社区/市场讨论**
- **风险点**

这样不会把“事实”和“市场解读”混在一起。

---

## 8. 当前实际限制

虽然 skill 已安装，但要真正可用，还需要你在各自 skill 目录中补 API Key 配置。

### `exa-search`
推荐配置位置：

- `skills/exa-search/config.local.json`

### `grok-search`
推荐配置位置：

- `skills/grok-search/config.local.json`

仓库里不应提交真实 key。

---

## 9. 最后的工作区规则

以后在这个工作区里，搜索问题默认这样判断：

- 如果用户说：
  - `官方怎么写`
  - `参数是什么`
  - `给我原文`
  - `找官网`
  - `抽正文`

  优先想到：`exa-search`

- 如果用户说：
  - `最近怎么了`
  - `大家怎么看`
  - `市场在讨论什么`
  - `X 上怎么说`
  - `舆情`

  优先想到：`grok-search`

- 如果任务涉及投研判断：
  - 先用 `grok-search` 看讨论面可以
  - 但最后必须用 `exa-search` 或官方源回证
